Publish Checklist

GitHub Pages公開前確認表

内容サンプル

AI生成記事や固定ページをGitHub Pagesへ公開する前に、事故を防ぐための確認表です。スターターキットに含める予定のテンプレートの一部を、サンプルとして公開しています。

外部リンク、購入ボタン、メール登録はありません。公開前の最終確認として必要な項目だけ使えます。

生成HTML確認

なぜ確認するか: 壊れた本文や空のページを公開しないため。

  • 新規記事HTMLまたは固定ページHTMLが存在する
  • 本文に「記事内容を読み込めませんでした」が残っていない
  • 必要なCSSが読み込まれている

index.htmlリンク確認

なぜ確認するか: トップページから新規記事へ自然に到達できるようにするため。

  • 新規記事URLがindex.htmlに含まれている
  • 旧節約記事やテスト記事がトップへ戻っていない
  • note-cardやAutomation Log表示が崩れていない

sitemap.xml確認

なぜ確認するか: 検索エンジンに現在の方針に合うURLだけを伝えるため。

  • 公開したい固定ページとAI自動化系記事が含まれている
  • 旧節約記事、テスト記事、壊れ記事が含まれていない
  • 日本語URLがpercent encodeされている

robots.txt確認

なぜ確認するか: sitemapの場所を伝え、noindexを読める状態にするため。

  • Sitemap行が公開URLを指している
  • 全体を不必要にブロックしていない
  • 旧記事整理はrobotsブロックではなくnoindexで行う

SEOタグ確認

なぜ確認するか: 検索結果やSNS共有で空の説明文を出さないため。

  • titleが空ではない
  • meta descriptionが空ではない
  • canonical、og:title、og:description、og:url、twitter:cardが入っている

noindex対象確認

なぜ確認するか: 旧テーマや壊れ記事を検索に出し続けないため。

  • 旧節約記事やテスト記事はsitemapから外れている
  • 必要な旧記事にはnoindex, followが付いている
  • 現在方針に合う記事へ誤ってnoindexを付けていない

Search Console認証ファイル保持確認

なぜ確認するか: 所有権確認状態を壊さないため。

  • site/google*.htmlを削除していない
  • 認証ファイルを通常ページのSEOチェック対象にしない
  • 認証ファイルの内容を別ファイルへ書き写さない

secret/APIキー混入チェック

なぜ確認するか: 公開repoへ認証情報を出さないため。

  • config.yaml、.env、token、credentialがcommit候補にない
  • 本文や固定ページにAPIキー、認証情報、個人情報を書いていない
  • __pycache__、.DS_Store、x_state.jsonがcommit候補にない

git diff確認

なぜ確認するか: 意図しない大量変更や旧記事復活を防ぐため。

  • 変更ファイルが今回の作業範囲だけになっている
  • 既存URL、slug、ファイル名を変えていない
  • git diff --checkが通る

commit/push確認

なぜ確認するか: 検証前の変更を公開しないため。

  • 検証が通ってからcommitする
  • site repoと親repoを混同していない
  • push前にcommit候補を再確認する

公開URL 200確認

なぜ確認するか: GitHub Pages反映後に読者が開けることを確認するため。

  • トップページが200で返る
  • 新規記事または固定ページURLが200で返る
  • sitemap.xmlとrobots.txtが公開URLで確認できる

この確認表は公開前の事故防止用です。公開結果、検索順位、収益を保証するものではありません。
